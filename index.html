继续快捷方式

大神们都再使用快捷键，你还在想什么

继续上次的话题，让我们继续说说更多的快捷键

Command+Shift+O

找方法，找函数，找各种就靠它了

Command+Shift+J

将文件目录切换到当然打开的文件

Command+J,->

将当前文件切换其他assistant editor中

Command+Alt

将m文件再主editor中打开

调试区的快捷键

除了Editor中使用的快捷键，再调试区也有很多非常实用的快捷方式，非常值得我们使用起来。
你有没有遇到这种情况，调试区已经输入n行信息，这时候你想删除它们，需要点击垃圾桶button，这时你只需要输入

Command+K

轻轻松松搞定啦

如果你想继续刚才打住的断点继续执行，直接输入

c+Enter

可以帮助你。

赶快试试吧，是不是发现非常简单好用呢。什么？还想知道更多的调试快捷键，输入h查看吧，只有你不知道的，没有你不想要的~什么查看线程，查看物理地址，都是分分钟搞定的事儿~

我记得层级有个大神告诉我，编程基本上使用键盘就可以了，键盘鼠标来回切换，累不累~希望大家多多给建议，看看大家都在使用什么快捷键。

在StackOverflow.找到了一个特别棒的关于快捷键的讨论，分享给大家

工欲善其事,必先利其器

单单使用Xcode，工作太单调，有很多插件可以让编程既有意思又高效起来。有了Xcode插件，编程也是如此美好。

终归一句话Lazy Code

http://alcatraz.io/打开这个之后，估计大家都会用了吧。再多啰嗦几句我必使用的插件吧。

FuzzyAutocompletePlugin

有了它之后，随便输入几个字母，相关的类、方法都显示出来了。

KSImageNamed

写完[NSImage imageNamed: 或者 [UIImage imageNamed:之后，图片通通显示出来。

ColorSense

同样写完[NSColor: 或者 [UIColor之后，要定义的颜色会显示出来。

VVDocumenter

上次介绍过的标注插件，真是不用就out了呀~

XAlign

让代码瞬间对齐，强迫者适用。

xcfui

可以找到没用的#import，代码提速有它的功劳。

VVDocumenter

XCode的快捷键们

介绍完插件之后，Xcode的快捷键也是开发中必须用起来的秘密武器。之前有个非常爱好iOS编程的伙伴告诉我说，没有快捷键我都不像编程了，那让我们看看快捷键能让编程变得多快捷吧。

Command 1~8

再Navigators中互相切换, Command 0 关闭navigation

Command Alt 1~6

在Inspectors互相切换,同理 Command Alt 0也时将它关闭掉

Control Command Alt 1~4

再Libraries中相互切换

Control 1~6

展示Jump Bar的下拉选项

Command + Enter:

显示standard editor

Command Alt Enter:

You can probably guess what this does. Indeed, It opens the assistant editor.

Command Alt Shift Enter:

打开revision control editor.

Command + Shift + Y.

打开或者关闭掉debugging area

还不知道这些快捷键的猿和媛们，快试试吧！或者你正在说这些快捷键太简单了（我听见你再说），好吧！下次我会分享一些Ninja Master才会的快捷操作，希望能对大家有用，期待下次的分享吧~

项目分享

这是小王爷的第一篇博客，首先简单的介绍一下我自己(此处有掌声)。我是一个iOS开发女攻城师。iOS编程大师和郭德纲都是我得男神(哈哈~)。有近4年iOS开发经验，现在在一家引以为豪的打车软件公司工作。我开通了微信公共账号，可以加我哦~如果有任何问题也可以给我发邮件到jack_gxy@126.com。欢迎进入iOS开发世界~

我的项目经验

我做过大大小小的App，上线的有7、8个了，我也是从一个小白慢慢走过来的，中间也是障碍不断，今天根据开发经验大致的总结一下。

命名一定要规范

减少缩写

不要自创缩写命名，缩写只用于通用专业术语，如URL等。宁可让名字长一些，团队中其他人容易理解。

过程化

将will，did，should将入命名中，会比较形象吧。

前缀，前缀

类名加上工程缩写
DDZMainViewController

常量加类型名前缀
UIApplicationDidFinishLaunchingNotification

枚举类型命名要加相关类名前缀，并且枚举值命名要加枚举类型前缀
typedef NS_ENUM(NSInteger, DBDStriveBarStatus){

DBDStriveBarStatusWaitStrive = 0,

DBDStriveBarStatusWaitGoing

};

给一个带修饰的对象命名时要采用修饰+类型的方式，而不是先指定其类型
UIButton *sendMsgButton；

使用stroyboards，xib

对使用storyboard、xib，还有只使用代码开发的争论一直不断，从我个人角度，我比较中意使用sb(只是缩写，没有其他意思)，xib。比较简单的视图，我们可以使用sb，xib，简化我们的开发。而且自从iOS8开始，我们可以对其进行个性化扩展，真是其乐无穷了。

第三方控件使用Cocoapods管理

我自己的开发经验，还有和其他大牛牛们讨论过，很多知名的APP都有使用Cocoapods，但是一些大公司会将第三方控件保存到自己的服务器中集中管理。

代码框架清晰

将Views，Model，Controllers，Resource，Network，Widges的文件夹建立的清晰一些，并且将成千上百的类分得明确一些。

Faux Pas

当我发现这个APP的时候，我马上更新了我的个性签名--妈妈再也不用担心我的编程。
Faux Pas是一个强大的辅助编程工具，它可以找出常见的错误，隐藏的bug，不良的编程习惯以及可维护性问题和风格问题。

Debug

当程序崩溃的时候，我们希望快速定位到崩溃点，一般的方法是通过dSYM文件分析。
现在我在贵公司发现了Crashlytics,它的原理其实就是根据dSYM文件出来，将物理内存翻译成代码展现给我们，并且个性化的操作可以及时通知。

公有Provisioning

我在之前的公司遇到这种情况，大家都是管理员身份，生成自己profie。每次更新完代码之后在setting中都要修改一下。后来我发现，大家建一个公共profile比较方便，再也不用修改setting中得profile，虽然更改起来不是很麻烦，但是只要是可以优化的方法，我们都要有常识的心态。

加注释

之前我的师傅经常和我说，加注释加注释。是因为我觉得很浪费时间，但是我发现别人的代码我真的需要一定时间来揣摩，大家是一个团队，你好我好大家好的原则，注释太重要了。猫神的VVDocumenter你还没有听说过吗？

每个类不要太冗余

我们被称为神一样存在的非主流iOS编程大师说过一句话，如果一个类超过300行，就该去面壁思过了，好吧~我之前的标准是500行，我和他的差距有200行那么多呀？！

今天就和大家说这么多吧，大家可以关注我的公众账号，也可以随时给我发邮件。
如果你对iOS有编程热情，如果你是iOS编程殿堂级大神，如果你还有那么一丢丢的帅(这个可以忽略)，如果你也和我一样是编程女孩子，那我们做朋友吧~
