封装思想，抽象一切

  • 实践经验

我不知道这样的命题符不符合我今天要分享的内容，大家看后评论。
你有没有在coding的时候遇到这样的情况，重复写一句或者一段代码，当遇到这种情况的时候，第一反应就是不将就，这样的重复代码不该出现再你的app中，马上应该封装个类或者方法，将这些方法抽象出来。

如何封装，怎样调用。废话少说，先看看我都封装了什么。

1.View视图中
在view中添加其他控件的时候需要以view的frame作为参考。在编写代码的时候一会儿一个view.frame.size.width烦不烦，好~封装这些方法。

  • 首先创建一个Class为UIView的Category
  • 再h文件中声明一个类型为float的width的变量
  • 在m中实现其get和set方法

-(CGFloat)width {
return self.frame.size.width;
}

  • (void)setWidth:(CGFloat)width {
    CGRect frame = self.frame;
    frame.size.width = width;
    self.frame = frame;
    }
  • 万事具备，只差使用。在其他类中只要引用这个类名，那么self.someview.width就清飘飘的可以使用了

介绍了width之后，我们还可以照猫画虎的实现很多方法，例如height，left，right，bottom，head等等。

2.safe判断
有时会遇到这样的情况，自己生成的或者服务器返回的数据类型不正确，造成app崩溃。所以这些经验告诉我们，做一些数据格式的判断还是有必要的。

-(BOOL)isValidString:(NSString )aString;
-(BOOL)isValidArray:(NSArray )aArray;
-(BOOL)isValidDictionary:(NSDictionary )aDictionary;
-(BOOL)isValidInteger:(NSInteger)count;
-(BOOL)isValidDistance:(NSString )aDistance;
-(BOOL)isValidTime:(NSString *)aTime;
-(BOOL)isValidCoor2D:(CLLocationCoordinate2D)aCoor2D;

3.关于各式各样Alert View

Alert View作为端的友好提示是程序中不可获取的一小部分，根据不同的提示场景，Alert View肯定会有不同，至少分为成功提示、错误提示、二次确认提示等等。Alert View大部分都是custom出来的，分为title、icon、content、cancel title、other title等等。所以我们可以封装一个custom alert view，将alert view的显示动画、消失动画定义出来，但是它的title、content等等留给调用它的类决定，再定义一个枚举方法，定义我们业务所需的各种各样的view样式，在custom alert类中，根据样式来显示不同的image。基本想法就是这些了，看看是怎么实现的。

  • 首先创建一个custom view，这个view中有两大部分组成，一个显示内容view，一个是背景view。
  • 内容view中有title label，icon image，content label，cancel button， other button，还有一个block。
  • 在view中实现显示，消失的动画效果等等，还有定义各种情景的alert，实现不同情景下得icon显示内容。

万事具备，只差调用，再h文件中写个调用的接口

-(void)showAlertWithTitle:(NSString)title message:(NSString)message defaultMessage:(NSString)message completion:(CompletionBlock)completion cancelTitle:(NSString) otherTitles:(NSString*)otherTitle;

这里，我多写了一个default message，其中message是网络请求之后，接口返回的message显示数据，如果接口返回失败，则使用default message。

4.table view翻页功能
基本任何app中都会遇到翻页需求，翻页就是给后台传基本参数外，还要传入当前请求页数和请求的条数。请求条数基本是个定值，但是页数是增加的，这也是有规律可循的，所以封装个方法配好翻页参数。
这个翻页的类要完成几个方法：上拉刷新、下拉加载更多和加载失败时的处理。

-(void)pullDownRefreshHandler:…

  • (void)loadMoreHandler:…
  • (void)finishRefreshWithError:(NSError *)error;
  • (void)finishLoadMoreWithError;

将翻页的功能交给这个类实现，不在请求中具体传入关于翻页的数据。
5.网络请求封装
大部分HTTP网络请求都是使用的AFNetwork吧，AF实在太好用，但是我们的需求是很多地方都要网络请求，在对AF进行一次封装太有必要。

  • 首先建一个HTTP请求类，将HTTP请求实现方法创建在此类中。将成功返回的数据和失败结果以block方式回调。
  • 各个controller有请求方法的时候创建其独有的Request Controller，在这个类中只需知道要传入什么参数，访问地址是什么即可，具体的HTTP请求方法则需之前的基类实现，看起来十分简单。

+(void)requestListWithDict:(NSDictionary*)dict completion:(completionBlock)completionBlock;

  • 接下来对应其ViewController直接调用这个方法即可，我们会发现这样的Request Controller会有很多个，因为它和ViewController中请求数对应，既容易查找也容易定位非常方便。

这些封装方法都是我们经常使用到得，当然这几种是远远不够的，在我们自己的产品中会慢慢积累很多封装的方法，比如加密类、JSON解析类、策略类、地图类、提示文案类等等等等。而且在这篇文章中，大家发现我只是介绍了一种思想但是没有具体实现任何一个方法，其实方法实现已经非常简单了，重要的是每当我们coding的时候，有没有时时刻刻想到封装和抽象呢。
你在实际的编程中遇到哪些比较好的封装类呢，我希望你分享给我。
